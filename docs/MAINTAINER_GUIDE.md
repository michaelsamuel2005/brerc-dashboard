# 🌿 Looking After the BRERC Dashboard — A Maintainer's Guide

**A gentle, plain-English guide for the person at BRERC who will look after the public dashboard once the student team has finished.**

> 💚 **Don't worry — you're in the right place.**
> You do **not** need to be technical to use this guide. You do **not** need to write any code. Read it slowly, one section at a time. Everything here is explained in plain words, and there's a glossary at the bottom for any word you're unsure about.

---

## What is this dashboard?

The **dashboard** is a page on the internet that shows BRERC's environmental and species records to **the public**.

In plain words:

- It is a **website**.
- Anyone can visit it — local residents, students, researchers, curious neighbours.
- It shows things like **maps of where species have been recorded** across the West of England, along with information and images about those species.

It was built by a student team from **180 Degrees Consulting Bristol** for **BRERC**, so that the public can explore BRERC's data in a friendly, visual way.

> 🧭 **Think of it like this:** BRERC holds a huge collection of nature records. The dashboard is a tidy, public "shop window" that lets ordinary people look at some of that collection safely and easily.

**Good news: much less than you might fear.** 🌱

You do **not** need to:

- ❌ Write any computer code.
- ❌ Understand how it was built.
- ❌ Fix it yourself when something goes wrong.

Looking after it mostly means being a **friendly caretaker**:

| Your job | What it means in plain words |
| --- | --- |
| 👀 **Check on it now and then** | Open the dashboard occasionally to make sure it looks right and loads properly. |
| 📣 **Share the link** | Give people the web address so they can visit. |

## How does it stay up to date?

The information you see on the dashboard comes from **BRERC's own records database** — the same collection of records BRERC already keeps. (A **database** is just a big, organised store of records; there's more on this in the glossary.)

- When BRERC's records change, the dashboard is designed to **reflect those changes**.
- You do **not** update the dashboard by hand. There is no typing in numbers or editing maps.

## How do I open and check it?

Checking the dashboard takes about a minute.

1. **Open a web browser.** This is the program you use to visit websites — for example **Google Chrome**, **Microsoft Edge**, **Safari**, or **Firefox**.
2. **Go to the dashboard's web address.**
   - The address is: **[TO BE CONFIRMED — the web address where the dashboard lives]**
   - You type it into the bar at the top of the browser (the **address bar**) and press the **Enter** key.
3. **Have a quick look.** Check that:
   - ✅ The page loads (you see the dashboard, not an error message).
   - ✅ You can see the **map**, and it has some content on it.
   - ✅ Species information and images appear where you'd expect.
4. **Share the link** with anyone who needs it: click once in the address bar to highlight the web address, copy it, and paste it into an email or message.

> 💡 **Tip:** You can save the web address as a **bookmark** (also called a "favourite") in your browser, so you don't have to type it each time. Look for a small ⭐ star icon near the address bar and click it. Once saved, you can return to the dashboard with a single click.

## Something looks wrong — what do I do?

Stay calm — most issues are small and easy to check. Work through whichever situation matches what you're seeing. **Do the simple checks first.**

### 🔄 The page won't load, or shows an error

1. **Refresh the page** — this means asking the browser to load the page again. Press the refresh/reload button (a circular arrow ↻ near the address bar).
2. **Try a different browser** — for example, if you were using Chrome, try Edge or Firefox instead.
3. **Check your own internet** — can you open another website, like a news site? If nothing loads, the problem is your internet connection, not the dashboard.
4. Still stuck? **Contact the help person** (see the next section).

### 🗺️ The map looks empty

1. This is **often not a fault**. An empty area on the map usually just means **there are no records for that spot** — which is completely normal for nature data.
2. Try **zooming out**, or **moving to a different area**, to see if records appear elsewhere.
3. If the **whole** map is blank everywhere, **refresh the page** (see above).
4. Still blank everywhere after refreshing? **Contact the help person.**

### 🖼️ An image or label is missing

1. **Refresh the page** — sometimes an image simply didn't load the first time.
2. If it's still missing after a refresh, it's worth reporting. **Note down which species or page** it was on, then **contact the help person.**

### 🐢 The site feels slow

1. **Check your own internet speed** — try loading another website to compare.
2. **Refresh the page** and give it a moment to load fully.
3. If it's **consistently** slow for you and others over several days, **contact the help person** and mention it.

> 🧡 **You can't break it by looking.** Refreshing, clicking around, and checking pages is completely safe. When in doubt, refresh first — it fixes a surprising number of things.

## Words you might see

Here are the everyday tech words used in this guide, in plain English.

| Word | What it means |
| --- | --- |
| **Browser** | The program you use to look at websites — e.g. Chrome, Edge, Safari, Firefox. |
| **Web address / URL** | The "location" of a page on the internet, like `www.example.com`. You type it into the browser's top bar. |
| **Address bar** | The wide box at the top of the browser where the web address appears and where you type it. |
| **Dashboard** | This project — the public web page that shows BRERC's maps and species information. |
| **Database** | BRERC's big, organised store of records. The dashboard reads from it but never changes it. |
| **Refresh (reload)** | Asking the browser to load the current page again — often the quickest fix for a small glitch. |
| **Bookmark (favourite)** | A saved shortcut to a web address, so you can return to it with one click instead of typing it. |
| **Hosting** | The service that keeps the website switched on and available on the internet, day and night. |
| **Screenshot** | A picture of what's on your screen. It's a handy way to show someone exactly what you're seeing. |
| **Accessibility** | Making sure people with disabilities (for example, those who can't use a mouse, or who use a screen reader) can use the website too. This is **legally required** — see below. |
| **Screen reader** | Software that reads a web page aloud, used by people who cannot see the screen. |
| **Bug** | A fault or mistake in the website that makes it behave wrongly. Reporting bugs is how they get fixed. |

## Please keep this safe 🔒

A few important things to remember:

- 🔑 **Never share passwords** or login details for anything connected to the dashboard. If someone asks you for them by email or phone, do **not** hand them over — check with the handover contact first.
- ♿ **The dashboard must stay usable by people with disabilities.** This is **not optional** — it is a **legal requirement** for public-sector websites in the UK (the standard is called **WCAG 2.2 AA**). If a change ever makes part of the site harder to use for someone with a disability, that's a serious problem — report it to the handover contact.
- 📄 **Keep this guide handy.** Save it, print it, or bookmark it. When something needs confirming, the **[TO BE CONFIRMED]** notes above will be updated with the real details at handover.

> 🌿 **Thank you for looking after this.**
> The dashboard exists to share BRERC's wonderful records with the public. By keeping an eye on it and reporting anything odd, you're helping it stay useful, welcoming, and open to everyone.
