# Contributing

The team's shared workflow. Keep it simple; agree any extra conventions together.

## Branch → pull request → merge
- Create a branch off `main`: `git switch -c <your-name>/<short-topic>`.
- Make focused commits with clear messages (what changed, and why).
- Push your branch and open a pull request.
- Have a teammate review before merging; keep `main` in a working state.

## Good habits
- Don't commit secrets or large/raw data files (see `.gitignore`).
- Pull `main` before starting new work so you stay in sync.
